# Security Policy

## Supported Versions

Only the latest active release version of BullDB is supported for security updates. 

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |
| < v1.0  | :x:                |

---

## Reporting a Vulnerability

We take the security of BullDB's field encryption, RLS injection layer, and multi-database routers very seriously. If you find a security vulnerability, please do **not** open a public issue. Instead, report it privately.

To report a vulnerability, please email the details to the project maintainers or use GitHub's private vulnerability reporting feature on the repository `https://github.com/vikukumar/bulldb.git`.

### Please include:
1. A description of the vulnerability and its potential impact.
2. Steps or proof-of-concept scripts to reproduce the issue.
3. Any suggested remediations or patches.

We will review your submission and respond within 48 hours to coordinate a fix and release a patch version.
